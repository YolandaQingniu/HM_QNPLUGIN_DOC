家用秤

概述

该文档介绍家用秤插件的接入，使用此插件前，必须先初始化基础鉴权插件

对接步骤

1、初始化家用秤插件

   //伪代码示例
   QNScalePlugin.initPlugin(qnPlugin)

2、按需设置家用秤插件的各种监听

    //伪代码示例

    //扫描监听器
    QNScalePlugin.scanListener = {
      onDiscoverDevice: (device: QNScaleDevice) => {
        AppLogger.logD(`发现 QNScaleDevice ${device}`)
      }
    }

    //状态监听器
    QNScalePlugin.statusListener = {
      onConnected: (device: QNScaleDevice) => {
        //设备已失败
      },

      onConnectFailed: (code: number, device: QNScaleDevice) => {
        //设备连接失败
      },

      onReadyInteractResult: (code: number, device: QNScaleDevice) => {
        //code == 0时，说明设备已允许交互
      },

      onDisconnected: (device: QNScaleDevice) => {
        //设备已断开
      }
    }

    //数据监听器
    QNScalePlugin.dataListener = {
      onGetRealTimeWeight: (weight: string, device: QNScaleDevice) => {
        //收到实时体重
      },
      onGetMeasureResult: (scaleData: QNScaleData, device: QNScaleDevice) => {
        //收到测量完成数据
      },
      onGetStoredData: (storedDataList: QNScaleData[], device: QNScaleDevice) => {
        //收到存储数据
      },
    }

    //设备用户管理监听器
    QNScalePlugin.userScaleEventListener = {
      onRegisterUserResult: (code: number, user: QNScaleUser, device: QNScaleDevice) => {
        //注册秤端用户结果
      },

      onSyncUserInfoResult: (code: number, user: QNScaleUser, device: QNScaleDevice) => {
        //同步秤端用户信息结果
      },

      onDeleteUsersResult: (code: number, device: QNScaleDevice) => {
        //删除秤端用户结果
        AppLogger.logE(`onDeleteUsersResultScale ${code}`)
      },

      onIdentifyWeightResult: (code: number, device: QNScaleDevice) => {
        //设置秤端用户识别体重结果
      }
    }    

    //事件监听器
    QNScalePlugin.eventListener = {
      onSetUnitResult: (code: number, device: QNScaleDevice) => {
        //设置秤端单位结果
        const result = (code == 0) ? "success" : "fail"
        this.measureViewModel.otherStatus = `SetUnit ${result}`
        AppLogger.logE(`onSetUnitResultScale  ${code}`)
      },
      onRequestLastDataHmac: (user: QNScaleUser, device: QNScaleDevice) => {

      },
      onSetFatMeasureSwitchResult: (code: number, device: QNScaleDevice) => {
        //设置秤端是否测脂结果
      }
    }

    //配网监听器
    QNScalePlugin.wifiListener = {
      onStartWiFiConnect: (device: QNScaleDevice) => {
        //开始配网
      },

      onConnectWiFiStatus: (code: number, device: QNScaleDevice) => {
        //配网结果
      },

      onDiscoveryNearbyWiFi: (ssid: string, rssi: number, device: QNScaleDevice) => {
        //扫描到周围wifi
      },
    }

3、开始扫描

正常开始扫描的前提是app已申请ohos.permission.ACCESS_BLUETOOTH权限且蓝牙开关是开启的

   //伪代码示例
   qnPlugin.startScan()

4、连接设备

如果开启扫描后，附近存在已开机的家用秤设备，会在扫描监听器的onDiscoverDevice方法收到发现的设备

   //伪代码示例

   //已发现设备，停止扫描
   qnPlugin.stopScan()
   //可以按需设置operate的属性，连接前调整想设置的单位
   const operate = new QNScaleOperate()
   operate.unit = QNScaleUnit.KG
   //连接发现的设备
   QNScalePlugin.connectDevice(device, operate)

5、状态监听

发起连接后，会在状态监听器收到相应的回调，app开发者可按需处理各种状态

6.1、正常测量

6.1.1、普通秤测量

通过设备对象的isSupportScaleUser方法，可以判断当前是否为用户秤

onReadyInteractResult回调设备允许交互后，设置测量用户
普通秤，调用setMeasureUser方法
随后在dataListener中可收到相应的数据回调

   //伪代码示例

   //根据实际情况设置用户的信息
   const user: QNUser | undefined = QNUser.buildQNUser(
     'xxxxxx',
     QNGender.MALE,
     30,
     172.0,
     false, {
     onResult: (code: number, msg: string) => {

     }
   })
   if (undefined != user) {
     QNScalePlugin.setMeasureUser(device, user)  
   }

6.1.2、用户秤测量

onReadyInteractResult回调设备允许交互后，设置测量用户
用户秤，调用setMeasureUserToUserDevice方法
可以将用户秤的测量场景分为以下三种情况

  • 首次使用注册秤端用户，将QNScaleUser的index设为0，visitorMode设为false，在秤上注册一个用户，注册结果会在onRegisterUserResult中回调,app需保存相关信息，以便下次连接时直接访问用户
  • 访问秤端用户，将QNScaleUser的visitorMode设为false，index和key设为SDK在onRegisterUserResult成功时回调的值，SDK会根据index和key访问秤上已注册的用户
  • 游客模式，游客模式是一种特殊的访问秤端用户，将QNScaleUser的visitorMode设为true即可，游客模式下连接的用户不会在秤上注册，因此用户秤在脱离蓝牙连接使用时，不会识别到游客模式下连接过的用户

随后在dataListener中可收到相应的数据回调

   //伪代码示例

   //根据实际情况设置用户的信息
   const user: QNScaleUser | undefined = QNScaleUser.buildQNScaleUser(
     'xxxxxx',
     QNGender.MALE,
     30,
     172.0,
     false, 
     0,
     1234,
     false, {
     onResult: (code: number, msg: string) => {

     }
   })
   if (undefined != user) {
     QNScalePlugin.setMeasureUserToUserDevice(device, user)  
   }

6.2、功能设置

在收到onReadyInteractResult回调时，如果code == 0，则说明设备已允许交互
此时可以调用一些功能设置方法，如设置秤端单位、删除秤端用户 等
最终相应的设置结果会在事件监听器eventListener或userScaleEventListener中回调

6.3、配网

如果家用秤支持扫描周围wifi，onReadyInteractResult回调设备允许交互后，调用scanNearbyWiFi方法，会在onDiscoveryNearbyWiFi收到周围wifi信息
传入配网信息进行配网，会在wifiListener中收到相应的回调

   //伪代码示例
   const info = new QNBPMachineWiFiInfo()
   info.ssid = ‘xxxxxxxx’
   info.pwd = ‘xxxxxxxx’
   info.serverUrl = ‘xxxxxxxx’
   //传入wifi信息，进行配网
   QNScalePlugin.startConnectWiFi(device, info)

7、断开连接

调用此方法，可断开已连接的设备

   //伪代码示例
   QNScalePlugin.cancelConnectDevice(device)
